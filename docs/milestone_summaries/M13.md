# M13 — Runtime surface coherence (internal error base + AuroraImage DRY)

**Timeframe:** Initialized 2026-03-28; merged to `main` 2026-03-29 (PR #14, merge `62f7128…`).

## Summary

M13 introduced **`AuroraRuntimeError`** as an **internal** base for **`SharedLibraryLoadError`**, **`ImageCreationError`**, and (later) audio errors; refactored **`AuroraImage`** with a private **`_from_dispatch`** shared path. **Lifecycle extraction** was explicitly **not** done — insufficient duplication to justify.

## What changed

- `errors.py`, `image.py` refactor, verifier list update, tests.

## What it proved

- Exception taxonomy could be **minimal and coherent** without changing public leaf constructor semantics.
- Coverage gates **remained** at 100% line/branch under the new structure.

## What it did not prove

- Exhaustive runtime error taxonomy for production MediaPipe use.

## Key deliverables

- `AuroraRuntimeError` module, DRY image creation path.

## Evidence (public)

- PR #14; CI success.

## Why it mattered

Coherent errors reduce debugging ambiguity as seams multiply (audio in M19).
