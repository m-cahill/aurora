# M14 — Dispatch operation token centralization (AuroraImage dispatch literals)

**Timeframe:** Initialized 2026-03-29; merged to `main` 2026-03-29 (PR #15, merge `b9d9f52…`).

## Summary

M14 added **`dispatch_tokens.py`** with **`IMAGE_FROM_FILE`** and **`IMAGE_FROM_BYTES`** string constants — **byte-identical** to pre-M14 literals — and wired **`image.py`** and tests to use them. Tokens are **not** re-exported from `aurora.runtime.__all__` (internal vocabulary).

## What changed

- New module, imports updated, value-stability tests for literal strings.

## What it proved

- Dispatch vocabulary has a **single source of truth** for the image seam.
- Refactors can preserve **string stability** with tests.

## What it did not prove

- Any native token recognition — only **in-repo** stability.

## Key deliverables

- `dispatch_tokens.py`, unittest pins for token values.

## Evidence (public)

- PR #15; CI green.

## Why it mattered

Scattered string literals drift silently; M14 made **seam vocabulary** reviewable and test-locked.
