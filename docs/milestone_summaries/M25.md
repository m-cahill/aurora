# M25 — ARB boundary framing and minimal run-bundle specification

**Timeframe:** Initialized 2026-04-01; merged to `main` 2026-04-02 (PR #26, merge `4037db3…`).

## Summary

M25 introduced **`docs/aurora_run_bundle_boundary.md`**: conceptual **AURORA Run Bundle (ARB)** boundary — offline/batch determinism narratives, **LIVE_STREAM** honesty, **certification external** (e.g., RediAI validates bundles; runtime does not embed certifier logic). Verifier and tests updated to require the new doc. **No** `src/aurora/` implementation.

## What changed

- Boundary doc; `aurora.md` references; verifier expectations.

## What it proved

- Artifact-bound execution could be **specified** as a **boundary concern** before stdlib ARB code landed.

## What it did not prove

- ARB on-disk implementation; semantic CI gates on bundle contents.

## Key deliverables

- `aurora_run_bundle_boundary.md` as normative framing for M26+.

## Evidence (public)

- PR #26; CI green.

## Why it mattered

ARB connects runtime narrative to **auditable artifacts**; M25 defined **what “bundle” means** at the program level.
