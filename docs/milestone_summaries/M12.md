# M12 — Branch coverage baseline + separate line/branch floors

**Timeframe:** Initialized 2026-03-27; merged to `main` 2026-03-28 (PR #13, merge `3bf18cf…`).

## Summary

M12 enabled **`branch = True`** in coverage and introduced **`scripts/check_coverage_thresholds.py`** to enforce **separate** line and branch floors from **`coverage.json`** — avoiding coverage.py’s **combined** fail-under pitfall when branches are measured. **`report:fail_under`** was removed from `.coveragerc` for that reason.

## What changed

- Threshold script, tests for the script, CI invocation after `coverage json`.
- `DEVELOPMENT.md` explanation of M11 vs M12 gates.

## What it proved

- **100% line and 100% branch** on the measured `src/aurora/` surface (as recorded at ship).
- Partial branches could be detected as failures via script logic.

## What it did not prove

- Exhaustive branch modeling for every future construct; **governance scripts** still out of coverage scope.

## Key deliverables

- `check_coverage_thresholds.py`, branch measurement discipline.

## Evidence (public)

- PR #13; ledger notes on branch arc counts at baseline.

## Why it mattered

Line-only gates can hide **untested branches**; M12 aligned the gate with **branch reality**.
