# M09 — Bounded domain smoke tests for current first-party runtime surfaces

**Timeframe:** Initialized 2026-03-27; merged to `main` 2026-03-28 (PR #10, merge commit `a7944ad…`).

## Summary

M09 added **composed** smoke tests: real **`SharedLibraryLoader`** with **patched** `ctypes.CDLL`, **`AuroraImage`**, and a **recording fake `Dispatcher`**. This validates **wiring** across modules — not isolated unit fakes alone — for happy paths and failure paths.

## What changed

- `tests/test_runtime_smoke.py`, documentation in `DEVELOPMENT.md` and seam framing.

## What it proved

- The **chain** from loader → dispatcher → domain type behaves as designed under test doubles.
- Failure propagation (`ImageCreationError` with chaining) at composed scope.

## What it did not prove

- Real library load, decode, or upstream Tasks parity.

## Key deliverables

- Composed smoke coverage for image seam (audio composition extended in M19).

## Evidence (public)

- PR #10; CI green per ledger entries.

## Why it mattered

Unit tests alone can miss **integration mistakes** at the seam; M09 closed that gap at the level **allowed** without real native binaries.
