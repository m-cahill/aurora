# M08 — Bounded image.py migration onto Dispatcher and loader seams (first-party AuroraImage)

**Timeframe:** Initialized 2026-03-27; merged to `main` 2026-03-27 (PR #9, merge commit `ba0110d…`).

## Summary

M08 added **`AuroraImage`** and **`ImageCreationError`**: image creation routes through injected **`Dispatcher.dispatch`** and **`LibraryLoader.shared_library`**. **`image.py` does not import `ctypes`.** Tests use fakes. This is the **in-repo** bounded image seam — **not** an upstream Tasks `image.py` migration inside a MediaPipe fork.

## What changed

- `image.py`, focused unit tests, verifier and doc updates.

## What it proved

- Domain construction can be **seam-only** without embedding loader details in the domain module.
- Failure paths preserve exception chaining via `ImageCreationError`.

## What it did not prove

- Decode correctness, MediaPipe parity, or real native execution on CI.

## Key deliverables

- `AuroraImage`, `ImageCreationError`, tests `test_image_runtime.py`.

## Evidence (public)

- PR #9; CI success on PR and post-merge `main`.

## Why it mattered

M08 demonstrated **behavior-preserving structure** at the Python boundary — the pattern later mirrored for audio (M19).
