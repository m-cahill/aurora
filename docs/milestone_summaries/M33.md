# M33 — Packaging metadata and open-source repository hygiene

**Timeframe:** Merged to `main` 2026-04-11 (PR #34, merge commit `335a1281…`).

## Summary

M33 made AURORA **installable** and **repo-clean** as an open package: **`pyproject.toml`** (setuptools, editable install, version **0.1.0**, **no** `[project.scripts]` — **`python -m aurora.arb`** remains the entry), **`src/aurora/py.typed`**, **`.python-version`** (**3.11**), **`CONTRIBUTING.md`**, **`SECURITY.md`**, and **`CODE_OF_CONDUCT.md`** (Contributor Covenant **2.1**). **`scripts/verify_repo_state.py`** gained guardrails against tracking workspace-private path shapes (`docs/prompts/`, `docs/manuals/`, `docs/milestones/`) and requires hygiene files including **`LICENSE`**. Docs (**`README.md`**, **`DEVELOPMENT.md`**, substrate notes) were aligned to **`pip install -e .`**.

## What changed

- Packaging and policy files; verifier expectations; documentation for local install.

## What it proved

- The repository could present a **standard Python project** layout while keeping **`src/aurora/`** behavior unchanged and **CI YAML** out of scope.

## What it did not prove

- New runtime correctness; moving dev pins fully into **`pyproject.toml`** (explicitly deferred where noted).

## Key deliverables

- **`pyproject.toml`** + **`py.typed`** + community docs; verifier **M33** hygiene list.

## Evidence (public)

- PR #34; post-merge **`main`** CI success IDs in [`aurora.md`](../aurora.md) (**Milestone M33**).

## Why it mattered

Downstream consumers need a **declared** install surface and hygiene signals; M33 provided that without conflating packaging with **native** or **MediaPipe** proof.
