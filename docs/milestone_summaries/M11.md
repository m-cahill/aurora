# M11 — Phase C kickoff: honest coverage baseline and first enforcement gate

**Timeframe:** Initialized 2026-03-27; merged to `main` 2026-03-28 (PR #12, merge `0ea5e37…`).

## Summary

M11 added **`coverage`** measurement for **`src/aurora/`** only, **`.coveragerc`**, and CI steps for **`coverage run` / `report` / `json`**. Protocol stub lines used **`# pragma: no cover`** so **100% line** enforcement on statements was honest. **Stdlib `unittest` only** — no pytest.

## What changed

- Coverage config, CI integration, `DEVELOPMENT.md` coverage section.

## What it proved

- First-party code could meet a **declared line floor** on `main` (100% after pragma alignment).
- Coverage scope intentionally **excludes** governance scripts from the same gate.

## What it did not prove

- Branch coverage depth (M12).
- Native or MediaPipe behavior.

## Key deliverables

- `.coveragerc`, coverage in `ci` workflow, pragma strategy for Protocol stubs.

## Evidence (public)

- PR #12; measured baseline and post-pragma numbers recorded in ledger.

## Why it mattered

**Honest coverage** is a contract with future refactors. M11 made regression on `src/aurora/` detectable.
